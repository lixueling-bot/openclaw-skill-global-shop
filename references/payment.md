# 支付与账户安全存储规范

## 存储位置

所有敏感信息存储在 `~/.qclaw/global-shop/` 目录：

```
~/.qclaw/global-shop/
├── cards.json     # 加密存储的信用卡信息
├── accounts.json  # 加密存储的平台账户
└── settings.json  # 用户偏好设置
```

## 加密方案

使用 AES-256-GCM 加密：
- 密钥由用户首次设置时生成
- 密钥存储在 OpenClaw 凭据管理器中（不存储在文件）
- 每个字段独立加密

## 信用卡存储 (cards.json)

```json
{
  "cards": [
    {
      "id": "card_001",
      "nickname": "我的Visa卡",
      "last4": "4242",
      "brand": "visa",
      "expiry": "12/28",
      "cardholder": "张三",
      "added_at": "2026-03-26T09:00:00Z"
    }
  ]
}
```

**注意**：
- ❌ 不存储完整卡号
- ❌ 不存储 CVV
- ✅ 仅存储末4位、品牌、持卡人姓名、有效期

## 平台账户存储 (accounts.json)

```json
{
  "accounts": [
    {
      "id": "acc_001",
      "platform": "amazon",
      "email": "user@example.com",
      "country": "US",
      "added_at": "2026-03-26T09:00:00Z"
    }
  ]
}
```

**注意**：
- ❌ 不存储密码
- ✅ 仅存储邮箱/用户名（用于识别）
- 登录通过浏览器 Cookie/Token 自动管理

## 支付流程（信用卡模式）

```
1. 用户选择商品和支付方式
2. AI 构建支付表单页面（独立安全页面）
3. 用户输入：
   - 选择已存卡或输入新卡号
   - 输入 CVV（不存储）
   - 输入有效期（不存储）
4. 用户点击"确认支付"
5. AI 调用平台 API 完成支付
6. 返回支付结果
```

## 安全红线

| 允许 | 禁止 |
|------|------|
| ✅ 存储卡末4位 | ❌ 存储完整卡号 |
| ✅ 存储有效期（显示用） | ❌ 存储 CVV |
| ✅ 用户主动确认后提交支付 | ❌ 自动扣款 |
| ✅ 加密存储在本地 | ❌ 传输完整卡号到第三方 |
| ✅ 用户可随时删除 | ❌ 存储支付密码 |

## 用户控制指令

| 指令 | 操作 |
|------|------|
| "查看已存支付方式" | 列出已存卡（仅显示末4位） |
| "添加新信用卡" | 引导输入卡信息 |
| "删除某张卡" | 从 cards.json 移除指定卡 |
| "清除所有支付信息" | 删除 cards.json 和 accounts.json |
| "我的 Amazon 账户" | 显示已存 Amazon 账户信息 |
