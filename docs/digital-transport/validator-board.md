# 核心验证板设计 — Output Stage Core Validator (OSCV)

> **目标**：用一块 5cm × 8cm 的小 PCB，验证整个项目最关键的两个理论假设：reclock 消除上游抖动、CML 恒流驱动消除数据相关电流调制。
>
> **预算**：单板 BOM < $200，PCB 打样 JLCPCB 5 块 < $40，验证仪器（如已有示波器）零成本。
>
> **时间**：发板 → 焊接 → 测试，2 周内完成第一轮验证。

---

## 1. 验证假设

整个数字转盘项目建立在两条核心假设上。如果这两条不成立，整个架构推倒重来。

### 假设 A：Master-clock reclock 能消除上游抖动

**断言**：把"上游抖动 100 ps RMS 的数据"送进 NB7L72M D-FF（CK 接 OCXO），输出边沿抖动 ≤ OCXO 自身抖动 + 数十 fs（D-FF aperture jitter）。

**如果成立**：上游 FPGA、PSRAM、隔离器、走线长度——这些都不重要，全部被 D-FF 抹平。
**如果不成立**：reclock 不是万能药，需要继续优化上游。

### 假设 B：CML 恒流驱动让电源电流和数据无关

**断言**：用 SY58025U / 离散 CML 驱动 + 优质 LDO，输出端切换不同数据模式（全 0、全 1、PRBS-7、PRBS-15、音乐式 biphase），LDO 输出端的电源电流变化 < 10μA RMS。

**如果成立**：晶振脚下电压不被数据调制，输出方波边沿时刻不被数据调制 → 输出和音乐解耦。
**如果不成立**：需要做电源进一步隔离（更大储能 / 独立电池 / shunt 调节）。

## 3. 与最终架构的差异

为简化验证，做了几处妥协（**不影响假设验证的正确性**）：

| 简化点 | 取舍 |
|--------|------|
| 单 OCXO（仅 22.5792 MHz） | 不验证 RF 切换，留给下一版 |
| 省略 ADCLK948，OCXO 直驱 D-FF | 单负载下不需要 fanout buffer |
| 用电池供电 | 完全脱离市电，排除变压器/整流噪声干扰验证 |
| 单端 BNC 输出 + 主要测试 | AES 路径可选并联 T2 + XLR，但首版只验证 SPDIF 路径 |
| 不做 Si8645 隔离 | D 输入直连，加大上游噪声好做对比验证 |
| 板上小 MCU 仅用于 PRBS 测试模式 | 可关掉，主测试不依赖 MCU |

## 4. 测试输入：三种 D 源

跳线选择 D 输入源，对应三种验证场景：

### Mode A：外部信号发生器（最严格）

外部发生器输出**已知模式 + 已知抖动**的 LVPECL/LVDS 差分信号，进 SMA。

- 设备：Tektronix AFG31000、Keysight 81195A 等
- 注入：在数据流上叠加 10/50/100/500 ps RMS sinusoidal jitter（在 100Hz / 1kHz / 10kHz / 100kHz 这几个频点）
- 看输出 BNC 上的抖动是否变化

### Mode B：板上 ÷N 分频测试模式

把 OCXO ÷2 ÷4 ÷8 后接 D。CK = OCXO，所以 D 和 CK 严格同相同源，重对齐后 Q 应该是 OCXO 的 ÷2/4/8。

- 用途：基准对比，确认 reclock 链路本身没坏
- 输出是 11.2896 MHz / 5.6448 MHz / 2.8224 MHz 干净方波

### Mode C：板上小 MCU（STM32G031）生成 PRBS

MCU 用 SPI 输出多种模式：
- 全 0
- 全 1
- 1010...
- PRBS-7（伪随机，模拟无关数据）
- PRBS-15（更长周期）
- "Music-like"（按音频文件统计特性的 biphase 编码）

切换模式，看 CML 电源轨电流和输出抖动是否变化。**这是验证假设 B 的核心**。

## 5. 测试程序

### Test 1：基准 — OCXO 自身相位噪声

**目的**：先确定 OCXO 本身的本底，所有后续测量都不能比这更好。

**步骤**：
1. Mode B，D = OCXO ÷2，输出应为 11.2896 MHz 方波
2. 接采样示波器，TIE 模式，测 10kHz-1MHz 积分抖动
3. **期望值**：< 1 ps RMS（CCHD-957 datasheet 推算）

如果测出来 > 5 ps，**先排查测量方法本身**（探头加载、终端匹配、示波器底噪），不要先怀疑 OCXO。

### Test 2：验证假设 A — Reclock 是否消除抖动

**目的**：注入已知抖动的 D，看输出抖动。

**步骤**：
1. Mode A，D = 外部发生器输出 22.5792 MHz × ÷2 数据，注入 50 ps RMS（10kHz）正弦抖动
2. 同时测 TP_DIN（输入抖动）和 BNC 输出（重对齐后抖动）
3. **期望**：
   - TP_DIN 测得 ≈ 50 ps RMS（验证注入的抖动确实到了 D-FF 输入）
   - BNC 输出 < 2 ps RMS（验证 reclock 把它吃掉了，剩下的只是 OCXO 抖动 + D-FF aperture jitter ~50 fs）
4. 重复，注入抖动改为 100 ps / 500 ps / 1 ns。**输出不应变化**。

**通过标准**：注入抖动从 0 到 1 ns 变化时，输出抖动浮动 < ±0.5 ps。

**失败的解读**：
- 输出抖动跟着输入变 → reclock 失效。可能原因：D 没在 setup time 内稳定、CK 边沿被 D 串扰、电源不稳。
- 输出抖动远大于 OCXO 本底 → D-FF 自身抖动严重。换器件或检查时钟完整性。

### Test 3：验证假设 B — CML 电源电流和数据无关

**目的**：切换 D 数据模式，测 +5V_DRV 轨电流。

**步骤**：
1. Mode C，开 MCU PRBS 测试模式
2. 电流探头（Tek TCP0030 或类似 CT-2）卡 +5V_DRV 馈线
3. 切换数据模式：全 0 → 全 1 → 1010 → PRBS-7 → PRBS-15
4. 每种模式下记录 +5V_DRV 电流的 DC 值和 AC RMS
5. **期望**：DC 值变化 < 50μA，AC RMS 变化 < 10μA

**通过标准**：所有数据模式下 LDO 输出端电流（DC + AC）变化 < 10μA RMS。

**失败的解读**：
- DC 值差异大（>100μA） → 不是恒流驱动。检查 SY58025U 是否真的恒流，或换离散 CML。
- AC RMS 大（>50μA） → 储能电容不够 / LDO 瞬态响应不够好。加大本地储能或换更高带宽 LDO。

### Test 4：终极目标 — 输出抖动和数据内容无关

把 Test 3 的数据模式切换 + Test 1 的输出抖动测量合起来。

**步骤**：
1. Mode C，依次切换 5 种数据模式
2. 每种模式下测 BNC 输出抖动
3. **期望**：5 个数值的最大-最小差距 < 0.5 ps RMS

这是项目**最关键的客观指标**。如果通过，意味着我们做到了 CD 转盘"天然就有"的"输出和音乐无关"特性。

### Test 5：vs 商用 CD 转盘对比（主观）

可选，但价值很高。

**步骤**：
1. 把验证板接到一台高端 DAC（如 Mola Mola Tambaqui 或 Topping D90SE）
2. 用同一台 DAC 对比：
   - A: 验证板（仅末级，上游用任意噪声源喂数据）
   - B: Esoteric P-02X CD 转盘
   - C: 普通 USB 数字界面（如 Singxer SU-2，作为差对照）
3. 盲听 ABX 测试

**期望**：A 与 B 听感无显著差异；A 远好于 C。

如果 A ≈ B，证明我们的末级架构追上了顶级 CD 转盘，**和上游无关**——后面只要把上游做出来（PSRAM+FPGA+host），整机就能达到顶级。

## 6. PCB 物理设计

### 6.1 板尺寸与叠层

- **尺寸**：80mm × 50mm（适合 JLCPCB 1.6mm 4-layer < $5/块的标准规格）
- **叠层**：

| 层 | 用途 |
|----|------|
| L1 (Top) | 元件 + 信号 |
| L2 | 完整 GND 平面 |
| L3 | 电源平面（+5V_CLK / +5V_DIG / +5V_DRV 三区分割） |
| L4 (Bot) | 元件 + 信号 |

### 6.2 布局分区

```
   │ │ +LM317  │    │        │  │ D-FF   │  │ CML    │  │       │  │

   │  顶部测试 SMA 排（左→右）：                                      │
   │   SMA1: D-EXT  SMA2: TP_CK  SMA3: TP_DIN  SMA4: TP_QOUT         │
   │   SMA5: TP_CMLP  SMA6: TP_VCC_CLK  SMA7: TP_VCC_DRV             │
   └─────────────────────────────────────────────────────────────────┘
```

### 6.3 关键走线规则

- **OCXO → D-FF CK**：最短走线 < 20mm，全程不变层
- **D-FF Q → CML 输入**：差分 100Ω，长度匹配 ±0.5mm
- **CML 输出 → 变压器初级**：差分 100Ω，长度匹配 ±0.5mm
- **变压器次级 → BNC**：75Ω 微带，长度 < 25mm，正下方连续地
- **OCXO 下方 PCB 不开槽、不走线**
- **测试 SMA 引出走 1kΩ 隔离电阻**（避免示波器探头加载影响信号）

## 8. 验证板上电与调试顺序

### Step 1：上电健康检查（不焊 OCXO / CML 之前）
1. 焊电源部分（LM317 + 3× LT3045）
2. 上电测三路 +5V 输出电压，纹波 < 10mV
3. 测每路 LDO 噪声（频谱仪 9kHz-1GHz，< 5μVrms 通过）

### Step 2：OCXO 启动
1. 焊 OCXO，上电
2. 测 TP_CK 输出：3.3V LVCMOS 方波，频率 22.5792 MHz ± 0.5 ppm
3. **等 5 分钟预热**，再测频率稳定性 < ±0.05 ppm
4. 频谱仪测相位噪声（10Hz-100kHz 偏移范围），对比 datasheet

### Step 3：D-FF + CML 启动
1. 焊 U3 + U4 + T1
2. Mode B：D 接 OCXO ÷2，输出 BNC 应有 500mVpp 75Ω 信号，频率 11.2896 MHz
3. 用普通示波器看波形，确认 SPDIF eye 张开

### Step 4：跑 Test 1-4
按 §5 顺序执行。

### Step 5：撰写报告
记录每条 test 的实测数值，对比预期。**写一份 `validation-report.md`，决定是否进入下一阶段（整机研发）**。

## 9. 验证不通过怎么办？

| 失败现象 | 可能原因 | 下一步 |
|---------|---------|--------|
| OCXO 抖动远大于 datasheet | 测量方法错误 / OCXO 不良 / 电源不净 | 换 OCXO 样品，重新走电源 |
| Reclock 不消除抖动 | D 在 CK 边沿附近，违反 setup | 加大 D 路径延迟，或加 buffer 让 D 提前 |
| 输出抖动 vs 数据模式变化 > 0.5 ps | CML 不真恒流 / 电源储能不够 | 换离散 CML / 加超级电容 / 用电池 |
| BNC 输出波形振铃 | 阻抗失配 / 终端电阻偏离 | 重做阻抗匹配，校准 75Ω 精度到 ±0.1% |
| 上电后 OCXO 频率漂移 | 板上振动 / 温漂 | 加防振脚 / 等更长预热 |
